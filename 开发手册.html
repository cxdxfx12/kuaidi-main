01 / OVERVIEW

项目概述

一款面向快递行业的专业派费结算系统，支持大规模Excel数据导入、智能规则匹配、多级结算分配，为物流企业提供完整的财务结算解决方案。

智能计费引擎

三层规则体系（客户级 → 区域级 → 全局级），支持14个省份分区，自动识别列名，精准计算派费。

高性能处理

安全认证

表现层 (UI)

基于PyQt5的桌面应用，包含登录窗口和主窗口。使用QThread实现后台任务，避免UI阻塞。支持文件预览、计算进度、结果展示、多级结算等功能。

业务层 (Services)

核心业务逻辑封装，包括：Excel解析、列名匹配、规则管理、费用计算、结算分配、数据导出。服务层依赖核心层完成计算。

核心层 (Core)

fee_calculator.py 实现三层规则匹配算法；settlement.py 实现多级结算引擎。两模块无UI依赖，可独立测试和复用。

数据层 (Models)

基于SQLAlchemy ORM的数据库模型，包含完整的表结构定义。database.py实现连接池管理和SQLite优化配置。

fee_calculator.py — 派费计算引擎

三层规则匹配算法，优先级：客户级(station) → 区域级(region) → 全局级(global)。支持重量进位模式和保底价。

fee_calculator.py 
class FeeCalculator:
    """派费计算器 - 核心计算逻辑"""

    def calculate(self, weight: float, region: str = "") -> Dict:
        # 1. 异常检查：无重量返回0
        if weight is None or weight <= 0:
            return {"fee": 0.0, "rule_name": "无效重量", "is_exception": True}

        # 2. 匹配规则 - 三层规则优先级匹配
        rule = self._match_rule(region)

        # 3. 计算派费 - 首重 + 续重
        first_weight = float(rule.get("first_weight", 1.0))
        first_fee = float(rule.get("first_fee", 5.0))
        continued_fee = float(rule.get("continued_fee", 2.0))
        min_fee = float(rule.get("min_fee", first_fee))

        if weight <= first_weight:
            fee = first_fee
        else:
            continued_weight = weight - first_weight
            fee = first_fee + continued_weight * continued_fee

        # 4. 保底价 - 确保每单最低收费
        fee = max(fee, min_fee)

        return {"fee": fee, "rule_name": rule.get("name"), "is_exception": False}

规则匹配逻辑

  • 按 priority 字段升序排列，数字越小优先级越高
  • 区域匹配支持关键词包含判断：规则关键词 in 区域文本
  • 未匹配任何规则时使用 default_rule 兜底
  • 保底价确保每单最低收费，避免极端低价

settlement.py — 多级结算引擎

支持四种结算维度：网点、快递员、承包区、月结客户。每种结算自动计算分成比例和收益。

settlement.py - 网点结算 
def calculate_station_settlement(self, details: List) -> List[Dict]:
    """按网点汇总结算"""
    # 分组key：网点编码 + 网点名称
    station_map = defaultdict(...)

    for d in details:
        if d.is_exception: continue
        key = (d.station_code, d.station_name)
        station_map[key]["total_fee"] += d.calculated_fee
        station_map[key]["order_count"] += 1

    # 分成比例：默认15%
    commission_rate = Decimal("0.15")
    commission = info["total_fee"] * commission_rate

    return [{
        "station_income": float(commission),
        "company_income": float(info["total_fee"] - commission)
    }]

快递员结算

快递员结算默认提成比例 80%，剩余20%归网点。支持按快递员分组统计订单量和派费总额。

承包区结算

承包区按网点编码前3位分组，默认分成比例 10%。用于大区层面的汇总统计。

月结客户结算

月结客户按"客户编码+客户名称"分组，包含代收货款(COD)统计。支持大客户月结账单导出。

规则定义 
class Rule:
    """单条计费规则"""
    def matches(self, region="", station_code="", weight=0.0) -> bool:
        # 网点匹配 - 客户专属规则优先
        if self.stations.strip():
            station_list = [s.strip() for s in self.stations.split(",")]
            if station_code not station_list: return False

        # 区域匹配（包含任一关键词即命中）
        region_match = any(k in region for k in keywords)

        # 重量匹配 - 重量区间过滤
        weight_match = self.min_weight <= weight <= self.max_weight

        return region_match and weight_match

SQLite 优化配置

def _apply_optimization_pragmas(connection):
    # WAL模式 - 写更快、读不阻塞写
    cursor.execute("PRAGMA journal_mode = WAL")
    # 同步策略 NORMAL - 比FULL快5-10倍
    cursor.execute("PRAGMA synchronous = NORMAL")
    # 页面缓存 200MB
    cursor.execute("PRAGMA cache_size = -200000")
    # 临时表放内存
    cursor.execute("PRAGMA temp_store = MEMORY")
    # mmap 1GB 内存映射
    cursor.execute("PRAGMA mmap_size = 1073741824")

08 / OPTIMIZATION

性能优化建议

针对大规模数据处理的深度优化指南，经过生产环境验证。

计算性能优化

  • 规则索引预构建，避免重复扫描
  • 多进程并行，充分利用 CPU 多核
  • 批量入库，25000 行/批 最优阈值
  • 延迟加载大文件，内存流式处理
  • 数据对象扁平化，减少 GC 压力

数据库优化

  • SQLite WAL 模式：读写分离，无阻塞
  • PRAGMA synchronous = NORMAL 平衡性能与安全
  • cache_size = -200000，约 200MB 页缓存
  • mmap_size = 1GB，内存映射加速读写
  • 使用事务批量写入，减少 IO 次数

代码结构建议

  • 核心计算模块与 UI 解耦，便于测试
  • fee_calculator / settlement 纯函数化
  • 配置文件与代码分离，易于二次开发
  • 添加单元测试，覆盖关键规则分支
  • 使用类型注解，提升 IDE 提示体验

安全与稳定

  • 账号密码使用 PBKDF2 + salt 哈希存储
  • 异常数据标记而非丢弃，便于追溯
  • 关键操作加锁防止并发写入冲突
  • 定期备份 SQLite 文件，提供恢复入口
  • 登录 Token 过期失效，免登录更安全
