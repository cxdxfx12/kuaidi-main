## 一、项目概述

- **软件名称**：大圣·快递物流出港账单结算系统
- **开发单位**：杭州喵喵至家网络有限公司
- **技术栈**：Python 3.8 + PyQt5 + pandas + SQLAlchemy
- **打包方式**：PyInstaller 单文件 EXE
- **核心功能**：Excel 导入 → 按网点/区域/客户规则计算派费 → 导出多级结算报告 → 历史记录管理

## 二、计算逻辑检查与优化

### 1. 计算引擎架构

```
Excel 数据
   ↓ (Python 读取，pandas / python-calamine)
calculate_service.CalculateService
   ↓
规则匹配（Station → Outlet → Region → Global 四级）
   ↓
费用计算（保底费 + 续重 + 活动加价）
   ↓
SQLite 持久化存储（fee_detail / fee_record）
   ↓
前端表格展示 + Excel 导出
```

### 2. 体积重与计费重量公式

```
体积重 = 长 × 宽 × 高 ÷ 计泡系数
计费重量 = max(实重, 体积重)
```

计泡系数标准值：
- **顺丰 / 京东 / 德邦**：6000
- **圆通 / 中通 / 韵达 / 申通 / 极兔 / 天天**：8000
- **EMS / 中国邮政**：5000
- **大件轻抛（优速/安能等）**：12000

### 3. 保底费与续重计算

| 重量区间 | 计算方式 |
|---------|---------|
| 0 ~ 首重 kg | 首重费用（若低于保底费则按保底费） |
| 首重 kg 以上 | 首重 + (计费重量 - 首重) × 续重单价 |
| 无重量信息 | 按"无重量默认价"计收 |

### 4. 活动加价规则

- **按业务日期判断**（而非当前系统日期），确保历史数据按当时的活动正确计费
- **限定省份**：支持中英文逗号分隔，如"浙江,江苏,上海"
- **支持多规则叠加**：多个活动规则可并列设置，按规则优先级匹配

## 三、优化实施记录

### 优化 1：多进程计算 — ProcessPoolExecutor

**问题**：旧版使用 `multiprocessing.Pool`，在 Windows + PyInstaller 打包后启动子进程失败，回退到单进程。

**解决方案**：改用 `concurrent.futures.ProcessPoolExecutor`，更稳定的 API。

修改文件：[`app/services/calculate_service.py`](file:///e:/excelbest/app/services/calculate_service.py)

```python
from concurrent.futures import ProcessPoolExecutor, as_completed

with ProcessPoolExecutor(max_workers=pool_size) as executor:
    future_to_chunk = {executor.submit(_process_chunk, chunk): idx
                       for idx, chunk in enumerate(chunks)}
    for future in as_completed(future_to_chunk):
        chunk_results = future.result()
        # ... 批量处理逻辑
```

### 优化 2：按快递公司差异化的计泡系数

**问题**：之前所有快递公司共用一个默认计泡系数（6000），导致顺丰以外的快递体积重计算偏大。

**解决方案**：在 `fee_rules.json` 中新增 `计泡系数_MAP` 字段，并在代码中按快递公司名称模糊匹配。

修改文件：
- [`data/config/fee_rules.json`](file:///e:/excelbest/data/config/fee_rules.json) — 新增 `计泡系数_MAP`
- [`app/services/calculate_service.py`](file:///e:/excelbest/app/services/calculate_service.py) — 新增 `_find_计泡系数()` 函数

```json
"计泡系数_MAP": {
  "顺丰": 6000, "SF": 6000,
  "京东": 6000, "德邦": 6000,
  "圆通": 8000, "中通": 8000, "韵达": 8000, "申通": 8000,
  "EMS": 5000, "中国邮政": 5000,
  "极兔": 8000, "天天快递": 8000,
  "优速": 12000, "百世": 8000,
  "跨越速运": 6000, "安能": 12000, "天地华宇": 12000
}
```

匹配逻辑：
1. 先尝试匹配 `courier_code`（精确匹配，如 "SF" → 6000）
2. 再尝试 `courier_name` 模糊匹配（如 "顺丰速运上海分公司" 包含 "顺丰" → 6000）
3. 回退到 `station_code` / `station_name` 匹配
4. 都没命中 → 使用全局默认值（用户可在"全局默认设置"中调整）

### 优化 3：活动日期按业务日期判断

**问题**：之前用 `datetime.now()` 判断活动期间，2026 年 11 月的快递单在 6 月运行时无法命中"双十一"活动。

**修复**：`_apply_promotion` 和 `match_rule_fast` 增加 `business_date` 参数，从 Excel 每行的业务日期读取。

---

### 优化 4：省份限定支持中英文逗号

**问题**：用户在界面输入"浙江，江苏，上海"（中文逗号）时，系统只认英文逗号，导致匹配失败。

**修复**：在 `_build_rule_indexes` 预解析时，先做 `replace("，", ",")` 统一转换，再按英文逗号切分。

## 四、PyQt5 选型说明

**当前环境**：
- Python 3.8.6
- PyQt5 5.15.9（Qt 5.15.2 LTS）
- PyInstaller 5.13.0

**为什么不选 PyQt6**：
| 维度 | 说明 |
|-----|-----|
| Python 版本 | PyQt6 要求更高，本项目定位 Python 3.8 兼容 |
| API 稳定性 | PyQt5 是长期稳定版本，社区资源丰富 |
| PyInstaller | PyQt5 的打包钩子更成熟、稳定 |
| 学习成本 | PyQt5 API 更直观、团队熟悉度高 |

**PyQt6 的优势**（未来如有需要可考虑升级）：
- 高 DPI / 4K 屏显示效果更好
- 原生支持系统暗色主题
- 部分新组件（如图表、多媒体）

**结论**：当前项目场景下 PyQt5 完全够用，无需为升级而升级。

## 五、性能分析

### 单次操作耗时

| 组件 | 单次开销 | 说明 |
|-----|---------|-----|
| 计泡系数匹配 | ~5 µs | 字典查找，可忽略 |
| 体积重计算 | ~1 µs | 简单算术运算 |
| 规则匹配 | ~10–20 µs | 四级优先级匹配 |
| **Excel 读取** | **主导** | 使用 python-calamine（Rust 引擎）提速 |
| **SQLite 写入** | **次级** | 每 5000 行批量提交 |

### 预估吞吐量

| 数据量 | 预估耗时 |
|--------|---------|
| 10,000 行 | 2–5 秒 |
| 50,000 行 | 5–12 秒 |
| 100,000 行 | 10–20 秒（触发多进程加速） |
| 500,000 行 | 40–80 秒 |

1. **Python-calamine（Rust 引擎）** 替代 openpyxl，Excel 读取提速 3–5 倍
2. **ProcessPoolExecutor**：10 万行以上自动多进程并行
3. **SQLite WAL 模式 + 批量写入**：5000 行提交一次
4. **计泡系数预加载到字典**：O(1) 查找
5. **规则索引预构建**：stations / regions / outlet 预索引

## 八、常见问题排查

### Q1：修改了"全局默认设置 → 计泡系数"，为什么计算结果没变？
**A**：保存后请重新点击"导入并计算"，规则重载后生效。如果还是不对，检查快递单的"快递公司名称"是否命中了 `计泡系数_MAP`（例如填了"顺丰"就会用 6000，不受全局默认影响 — 全局默认只在完全匹配不到快递公司时生效）。

### Q2：活动加价规则设置了但没生效？
**A**：检查两点：
1. 活动期间是否包含快递单的**业务日期**（不是当前系统日期）
2. 限定省份是否填写正确 — 中英文逗号都支持，但如果省份名称不完全匹配（如"浙江" vs "浙江省"）需要调整

### Q3：桌面图标还是模糊 / 显示旧图标？
**A**：执行一次"清理 Windows 图标缓存"步骤（见上文第六章），或把 exe 换个目录/文件名再试。

### Q4：如何测试差异化计泡系数是否生效？
**A**：准备一个简单的测试 Excel，把不同快递公司的订单放在一起（比如顺丰、圆通、EMS 各一行），计算后查看每条的体积重是否符合预期：
- 顺丰（6000）：长30×宽30×高30 → 体积重 = 4.5 kg
- 圆通（8000）：长30×宽30×高30 → 体积重 = 3.375 kg
- EMS（5000）：长30×宽30×高30 → 体积重 = 5.4 kg
